# Bead Description Template

Use this structure inside `br create --description`.

## Task
- One sentence: what to do.

## Capture reference
- Capture file: <absolute or repo-relative path to captured X-post/task>

## Background / Why
- Why we’re doing this; context.

## Scope
- In-scope:
- Out-of-scope:

## Key files
- `path/to/file.ext` — purpose

## Implementation notes
- Concrete approach, APIs/CLIs, gotchas.

## Acceptance criteria
- Bullet list of verifiable outcomes.
- Include commands to verify where possible.

## Edge cases / risks
- Security, performance, backwards compat.

## Rollback
- How to revert safely.
